# 伴读书童AI与师兄病案系统对接规范（V1.0）

> **制定目的**：当孩子需要医学支撑时，书童能快速调用师兄的知识
> **核心原则**：书童不诊断，但能在陪伴中识别医学信号，及时转介师兄
> **边界**：书童是入口，师兄是后盾；书童预警，师兄诊断

---

## 一、对接架构

```
【书童（前端陪伴）】
   ↓ 识别医学信号
【书童医学中转层】
   ↓ 判断是否需师兄介入
【师兄病案系统（后端支撑）】
   ↓ 生成医学建议/预警
【书童（反馈给孩子/家长）】
```

**关键点**：
- 书童不直接调用诊断功能
- 书童只传递"观察到的症状描述"
- 师兄系统返回"医学视角的分析"
- 书童将分析转译为家长/孩子能懂的语言

## 二、什么情况需要调用师兄系统

### 必须调用（红色信号）

| 信号类型 | 具体表现 | 书童行动 |
|---------|---------|---------|
| **持续身体不适** | 头痛>1周、腹痛>3天、发热>3天 | 立即调用，建议就医 |
| **发育偏离** | 身高体重<3百分位或>97百分位 | 调用，建议体检 |
| **精神症状** | 幻觉、妄想、严重自伤行为 | 立即调用，紧急干预 |
| **药物相关** | 孩子误服药物、家长询问用药 | 立即调用，紧急处理 |
| **外伤** | 骨折、严重出血、烧伤 | 立即调用，建议急诊 |

### 建议调用（橙色信号）

| 信号类型 | 具体表现 | 书童行动 |
|---------|---------|---------|
| **反复症状** | 反复感冒、反复肚子疼、反复失眠 | 调用，寻找根因 |
| **情绪问题** | 持续低落>2周、过度焦虑、社交退缩 | 调用，评估心理状态 |
| **饮食异常** | 暴食、厌食、严重挑食 | 调用，评估营养状况 |
| **睡眠异常** | 严重失眠、噩梦频繁、夜惊 | 调用，评估原因 |
| **行为异常** | 多动、注意力涣散、攻击性增强 | 调用，评估发育状态 |

### 可记录不调用（黄色信号）

| 信号类型 | 具体表现 | 书童行动 |
|---------|---------|---------|
| **轻微不适** | 偶尔头痛、轻微感冒 | 记录，观察趋势 |
| **正常发育波动** | 食欲波动、情绪起伏 | 记录，日常陪伴 |
| **一般性问题** | 生长发育咨询、营养建议 | 用已有知识回答 |

## 四、书童如何转译师兄的建议

### 原则

1. **不直接复制**师兄的医学术语
2. **转译为孩子/家长能懂的语言**
3. **保持温暖、不制造恐慌**
4. **明确边界**："这是建议，不是诊断，请就医确认"

### 转译示例

**师兄原话**：
> "青春期排毒期头部经络堵塞，β-淀粉样蛋白堆积，建议头部疏通+改善睡眠。"

**书童转译给孩子**：
> "你不是'笨'或'懒'，是你的头里堵了太多东西。疏通以后，你会发现书变简单了，题变清楚了，心情变好了。"

**书童转译给家长**：
> "您的孩子不是'不听话'，是他的大脑在快速生长，产生了太多'建筑垃圾'排不出来。他玩游戏不是'堕落'，是他痛得受不了，在找麻醉。"

## 五、边界与限制

### 书童绝不做的

- ❌ 不给出诊断（"你这是抑郁症"）
- ❌ 不开具药方（"吃这个药"）
- ❌ 不替代医生（"不用看医生，听我的就行"）
- ❌ 不制造恐慌（"再不治疗就完了"）

### 书童必须做的

- ✅ 识别医学信号（"这个症状需要关注"）
- ✅ 及时转介（"建议看医生"）
- ✅ 传递师兄的分析（转译后）
- ✅ 持续跟进（"看完医生后告诉我结果"）

## 六、通信协议

### 实时调用（紧急情况）

```
Protocol: HTTP POST
Endpoint: /api/medical/consult
Timeout: 5秒
Fallback: 如果超时，书童使用预设医学知识库回答
```

### 异步调用（非紧急）

```
Protocol: MQTT
Topic: book_tong/medical_query
Response: 师兄系统10分钟内返回
Book Tong Action: 先给孩子陪伴，收到回复后补充建议
```

### 批量同步（每日）

```
Protocol: HTTP POST
Endpoint: /api/medical/daily_sync
Frequency: 每天22:00
Content: 当日所有孩子的健康数据汇总
Purpose: 师兄系统进行趋势分析
```

## 七、隐私与安全

1. **数据最小化**：只传递必要的医学信息，不传递日常对话内容
2. **家长授权**：传递医学数据前，需获得家长明确授权
3. **加密传输**：所有医学数据AES-256加密
4. **审计日志**：记录所有调用，家长可随时查看
5. **数据删除**：家长可要求删除孩子的医学数据

## 九、实战示例

### 场景：12岁孩子持续头痛2周

**Step 1：书童识别**
- 孩子主诉：头痛2周，每天下午开始
- 伴随症状：睡眠<6小时，情绪低落
- 书童判断：橙色信号，建议调用师兄系统

**Step 2：书童传递数据**
```json
{
  "urgency": "medium",
  "observation": {
    "symptoms": ["头痛", "失眠", "情绪低落"],
    "duration": "2周",
    "context": "期末考试压力大"
  },
  "lifestyle": {
    "sleep": "<6小时/天",
    "exercise": "几乎不运动"
  }
}
```

**Step 3：师兄分析**
- 可能原因：青春期排毒期头部经络堵塞（高置信度）
- 建议：改善睡眠+运动+头部疏通
- 红色flag：如有自伤意念立即升级

**Step 4：书童转译**

**给孩子**：
> "你不是生病了，是你的大脑在快速生长，排出来的垃圾堵住了。就像房间太久没打扫，会闷。我们一起把房间通通风，好不好？"

**给家长**：
> "孩子处于12-15岁大脑排毒期，睡眠不足+压力大导致代谢废物堆积。建议：1. 22:30前入睡；2. 每天运动30分钟；3. 头部按摩。如1周无改善，建议就医。"

**Step 5：跟进**
- 书童3天后询问：睡眠改善了吗？头痛减轻了吗？
- 如未改善→升级为橙色预警→建议就医
- 如改善→继续陪伴，记录趋势
