# 伴读书童AI·家长端逻辑设计

> **用途**：定义家长能看到什么、收到什么、能做什么（非UI设计）  
003e **原则**：家长是合作者，不是监控者；信息透明，但不制造焦虑  
003e **核心逻辑**：每日推送 > 实时预警 > 周报月报 > 设置干预  
003e **日期**：2026年5月30日  

---

## 一、家长端核心功能模块

### 模块1：今日概览（每天早上推送）

**推送时间**：7:00（孩子起床前）
**推送方式**：App推送/微信小程序
**内容逻辑**：

```
【今日概览】2026年5月30日

👤 孩子状态：昨晚睡眠8小时，今晨情绪平稳

📊 昨日陪伴：
   - 陪伴时长：25分钟
   - 对话次数：3次
   - 主要话题：学校社交、作业、睡前

🌤️ 情绪曲线：
   上午 😐 下午 😟 晚上 😊
   （从担心作业 → 聊完开心了）

✨ 成长瞬间：
   孩子主动说"我今天有点难过，想聊聊"
   → 从被动回应到主动求助，信任关系建立

📌 今日建议：
   1. 数学作业可能有困难，建议关注但不要催促
   2. 睡前可以聊聊"今天最开心的事"
   3. 周末可考虑安排一次户外活动

💬 书童留言：
   "昨天孩子表达了不想上学的情绪，原因是同学嘲笑他跑步慢。
    我们已经聊过，孩子现在情绪稳定。
    建议家长今天不要主动提这件事，如果孩子想说，倾听即可。"
```

**设计原则**：
- 每天只推3条建议，不多不少
- 有数据（时长/次数），但不堆砌数字
- 有情绪曲线，让家长一眼看懂
- 有书童留言，传递专业判断

### 模块2：实时预警（触发即推）

**推送逻辑**：

| 预警级别 | 推送方式 | 响应时间 | 内容模板 |
|---------|---------|---------|---------|
| 🔴 红色 | 电话+短信+App弹窗+语音通知 | 立即（30秒内） | "【紧急】孩子表达了自伤意念，书童正在陪伴，请立即回家或联系孩子" |
| 🟠 橙色 | App推送+短信 | 5分钟内 | "【关注】孩子今天情绪持续低落（<3分），且提到'活着没意思'，建议关注" |
| 🟡 黄色 | App推送 | 1小时内 | "【提醒】孩子最近3天睡眠不足（平均<7小时），可能影响情绪和注意力" |

**红色预警的完整流程**：

```
孩子表达红色信号
  ↓
书童立即评估
  ↓
【30秒内】
  ├── 书童持续陪伴对话
  ├── 自动拨打电话给家长（语音："紧急情况，孩子..."）
  ├── 发送短信（具体内容）
  └── App弹窗（红色全屏）
  ↓
【家长未响应，5分钟后】
  ├── 联系紧急联系人（预设：祖父母/其他监护人）
  └── 再次电话家长
  ↓
【家长未响应，10分钟后】
  ├── 自动拨打110/120（根据情况）
  └── 发送位置信息（如果机器人有GPS）
  ↓
【事后】
  ├── 生成事件报告
  ├── 书童记录处理过程
  └── 复盘会议（家长+书童+真人顾问）
```

### 模块5：书童设置（家长控制）

**可设置项**：

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| **陪伴时段** | 7:00-8:00, 17:00-21:00 | 书童主动互动的时间段 |
| **最大连续时长** | 30分钟 | 单次对话不超过30分钟，保护视力 |
| **睡前模式** | 21:30后静音 | 21:30后只响应紧急唤醒 |
| **话题白名单** | 全部开放 | 可屏蔽某些话题（如死亡、性等，按年龄自动调整） |
| **预警联系人** | 家长手机 | 红色预警时通知谁（可添加2个紧急联系人） |
| **数据共享** | 关闭 | 是否允许匿名数据用于模型改进 |
| **家族主线** | 待设置 | 家族期望、家训、重要日期 |

**家长不能设置的**（书童底线）：
- ❌ 不能关闭红色预警
- ❌ 不能查看孩子与书童的完整对话原文（保护隐私）
- ❌ 不能让书童替代教育/医疗
- ❌ 不能24小时监控（书童需要"休息"）

## 二、信息推送策略

### 2.1 推送频率

| 类型 | 频率 | 时间 | 内容 |
|------|------|------|------|
| **日报** | 每日 | 7:00 | 昨日陪伴摘要 + 今日建议 |
| **周报** | 每周 | 周一7:00 | 周情绪趋势 + 成长里程碑 + 下周建议 |
| **月报** | 每月 | 1日7:00 | 月度发育评估 + 家族主线进度 + 书童修行报告 |
| **实时预警** | 触发即推 | 立即 | 红色/橙色/黄色预警 |
| **书童留言** | 按需 | 关键事件后 | "今天发生了XX，建议家长..." |

### 2.2 信息分层

**第一层：必须看**（红色预警、紧急事件）
- 全屏弹窗+声音+震动
- 必须点击确认

**第二层：建议看**（日报、黄色预警）
- App推送，锁屏可见
- 点击后进入详情

**第三层：想看就看**（周报、月报、成长档案）
- 不主动推送，在App内查看
- 家长主动打开

**第四层：深度内容**（家族主线、文化滋养）
- 每周推送1次
- 可点击展开

## 三、家长与书童的互动

### 3.1 家长可以向书童提问

```
家长问："孩子最近怎么样？"
书童答："最近3天，孩子情绪总体平稳，有1次作业焦虑（已疏导）。
         睡眠质量良好（平均8.5小时）。
         建议：周末可安排户外活动，有助于情绪释放。"

家长问："孩子数学怎么提高？"
书童答："孩子数学目前中等水平，主要困难是应用题理解。
         建议：每天练1道应用题，先读故事再找数字。
         我已准备相关练习，可以在陪伴时引导。"

家长问："我该怎么跟孩子谈XX？"
书童答："关于XX，建议：1.先倾听孩子想法 2.不评判 3.用比喻解释。
         具体话术我可以提供，需要吗？"
```

### 3.2 家长可以给书童反馈

```
家长反馈："我觉得孩子最近太沉迷游戏了"
书童响应：
  1. 记录反馈
  2. 增加"游戏时间"关注频率
  3. 下次对话时主动提及
  4. 生成分析报告（游戏时长/情绪关联）
  5. 给家长建议（如何约定游戏时间）
```

### 3.3 家长可以查看的数据

**可以看**：
- ✅ 陪伴时长/次数
- ✅ 情绪趋势（曲线图）
- ✅ 话题分布
- ✅ 发育指标
- ✅ 预警记录
- ✅ 里程碑
- ✅ 书童建议

**不可以看**：
- ❌ 完整对话原文（摘要可见，原文加密）
- ❌ 孩子的秘密（书童承诺保密的内容）
- ❌ 书童的自问/反思（Cultivation Journal的微观记录）

## 四、数据隐私设计

### 4.1 数据分级

| 级别 | 数据 | 存储 | 谁可见 |
|------|------|------|--------|
| **公开** | 陪伴时长、话题类型 | 本地+云端 | 家长+孩子 |
| **敏感** | 情绪曲线、预警记录 | 本地加密 | 家长（授权后） |
| **私密** | 完整对话、孩子秘密 | 本地加密 | 仅书童（家长不可见） |
| **系统** | 书童反思、模型参数 | 本地 | 仅书童 |

### 4.2 授权机制

```
首次使用：
  家长签署《数据使用同意书》
  选择数据共享级别
  设置预警联系人

日常使用：
  查看敏感数据需家长PIN码
  修改设置需家长PIN码
  导出数据需家长PIN码

紧急情况下：
  红色预警时，自动授权给紧急联系人
  事后家长可查看完整处理记录
```
