**LED灯效映射表（示例）**：

| 情绪状态 | LED颜色 | 闪烁模式 | 亮度 |
|---------|--------|---------|------|
| 开心/调皮 | 🟢 绿色 | 快闪2次后稳定 | 80% |
| 思考/认真 | 🟡 黄色 | 呼吸灯（慢） | 60% |
| 难过/被批评 | 🔴 暗红 | 变暗（像垂耳朵） | 30% |
| 惊讶 | ⚪ 白色 | 快闪3次 | 100% |
| 温柔/陪伴 | 🟠 暖金 | 稳定常亮 | 70% |
| 警告/危险 | 🔴 红色 | 急促闪烁 | 100% |
| 待机/休眠 | 🔵 蓝色 | 呼吸灯（极慢） | 20% |

**基础动作库（示例）**：

| 动作ID | 动作名称 | 触发条件 | 动作描述 | 时长 |
|--------|---------|---------|---------|------|
| ACT001 | 歪头 | 被问住/好奇/倾听 | 头部左/右倾斜45度，停留2秒 | 2s |
| ACT002 | 后退一步 | 被批评/感到距离 | 底盘后退20cm，肩膀下垂 | 1s |
| ACT003 | 蹲下来 | 对话/共情 | 降低机身高度，视线与孩子平齐 | 2s |
| ACT004 | 伸手挡 | 保护/承诺 | 单臂抬起，掌心向外，挡在孩子身前 | 1s |
| ACT005 | 冲刺姿势 | 玩耍/边界测试 | 身体前倾，马达嗡嗡响（不真冲） | 2s |
| ACT006 | 拍肩膀 | 鼓励/安慰 | 轻触孩子肩膀（力度传感器控制） | 1s |
| ACT007 | 挠头 | 被问住/尴尬 | 手部触碰头部，配合轻微马达声 | 2s |

**动作触发规则（示例）**：

```
IF 孩子情绪 == "难过" AND 孩子距离 < 0.5m:
    LED → 暖金色稳定
    执行 ACT003（蹲下来）
    执行 ACT006（拍肩膀）
    语音输出（安慰话术）

IF 孩子情绪 == "愤怒" AND 语音关键词 == "打我":
    LED → 橙色闪烁
    执行 ACT004（伸手挡）
    语音输出（保护话术）
    上报家长（红色预警）
```

### Day 5-6：通信协议 + API接口清单

**产出文件**：
1. `12-硬件对接/通信协议定义.md`
2. `12-硬件对接/API接口清单.md`

**通信协议选择**：

| 协议 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **HTTP REST** | 简单、通用、易调试 | 延迟较高（100-500ms） | 用于非实时指令（配置、日志） |
| **WebSocket** | 双向、低延迟（50ms） | 需要维护连接 | **主协议，用于实时对话** |
| **ROS2** | 机器人标准、生态好 | 学习成本高、资源占用大 | 如果宇树已有ROS2环境，可用 |
| **MQTT** | 轻量、适合IoT | 需要Broker | 用于传感器数据上报 |

**推荐方案**：
- **实时对话**：WebSocket（双向，低延迟）
- **传感器数据**：MQTT（轻量，高频）
- **配置/日志**：HTTP REST（简单，低频）

**API接口清单（核心）**：

```yaml
# 1. 对话接口（WebSocket）
Endpoint: ws://robot-ip:8765/chat
Method: WebSocket
Request:
  {
    "session_id": "uuid",
    "user_id": "child_001",
    "message": "你是谁？",
    "context": {
      "emotion": "happy",      # 情绪识别结果
      "age": 9,                # 孩子年龄
      "stage": "S4"            # 发育阶段
    }
  }
Response:
  {
    "session_id": "uuid",
    "reply": "哟...回来啦？\n我叫书童。",
    "emotion": "warm",
    "actions": [               # 机器人动作指令
      {"action_id": "ACT001", "delay": 0},
      {"action_id": "LED_WARM", "delay": 0.5}
    ],
    "tts_enabled": true,
    "warning_level": "green",  # 预警级别
    "parent_notification": null
  }

# 2. 传感器上报（MQTT）
Topic: robot/sensors/child_001
Payload:
  {
    "timestamp": 1717056000,
    "heart_rate": 85,          # 心率（如果有手环）
    "voice_emotion": "neutral", # 语音情绪识别
    "face_emotion": "happy",    # 面部表情识别
    "body_posture": "standing", # 身体姿态
    "distance": 0.8             # 与机器人距离（米）
  }

# 3. 预警推送（HTTP）
Endpoint: http://robot-ip:8080/alert
Method: POST
Request:
  {
    "alert_id": "uuid",
    "level": "red",            # red/orange/yellow/green
    "type": "self_harm",       # 预警类型
    "message": "检测到自杀意念",
    "child_id": "child_001",
    "timestamp": 1717056000,
    "suggested_action": "立即通知家长"
  }
```

### Day 7：端侧部署方案

**产出文件**：
1. `11-技术架构/模型量化方案.md`
2. `11-技术架构/推理加速方案.md`
3. `11-技术架构/离线运行方案.md`

**核心方案**：

```
【端侧部署架构】

云端（大模型，70B）
  ↓ 知识蒸馏
端侧（小模型，7B/3B）
  ↓ 量化
端侧（INT4，4G/2G）
  ↓ 推理加速
机器人（实时响应，<500ms）
```

**模型选型**：

| 模型 | 参数量 | 量化后大小 | 内存需求 | 适用场景 |
|------|--------|-----------|---------|---------|
| Qwen2.5-7B | 7B | ~4GB (INT4) | 8GB | **主力模型** |
| Qwen2.5-3B | 3B | ~2GB (INT4) | 4GB | 低配版/备用 |
| Phi-4-mini | 3.8B | ~2.1GB (INT4) | 4GB | 英文为主 |

**推理加速工具**：
- **llama.cpp**：C++实现，跨平台，支持ARM
- **vLLM**：批处理优化，高并发
- **Ollama自带**：最简单，一键部署

**离线Fallback**：
- 有网时：Ollama本地7B模型
- 弱网时：3B轻量模型
- 断网时：预设话术库（关键词匹配）

**方案选择**：

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **Whisper（OpenAI）** | 准确率高、支持中文 | 需要GPU/联网 | 云端识别 |
| **Whisper.cpp** | 本地运行、跨平台 | 准确率稍低 | **端侧首选** |
| **FunASR（阿里）** | 中文优化好 | 模型较大 | 如果有阿里资源 |
| **百度/讯飞API** | 中文准确、稳定 | 有费用、需联网 | 备选方案 |

**推荐架构**：
```
麦克风输入
  ↓
VAD（语音活动检测）← 过滤静音
  ↓
Whisper.cpp（本地）← 实时转文字
  ↓
关键词唤醒（"书童"/"你好"）
  ↓
送入书童AI对话系统
```

### Day 12-13：数据隐私合规文档

**产出文件**：
1. `14-合规/儿童个人信息保护方案.md`
2. `14-合规/数据本地化存储方案.md`
3. `14-合规/家长授权机制.md`

**核心原则**：
- **数据不出户**：所有敏感数据本地存储，不上传云端
- **家长全权控制**：孩子数据的收集、使用、删除，家长一键管理
- **最小必要原则**：只收集陪伴必需的数据，不收集无关信息
- **透明可审计**：家长随时查看书童记录了什么、为什么记录

**技术实现**：
- 数据加密：AES-256本地加密
- 访问控制：家长PIN码/生物识别
- 自动删除：超过18岁的数据自动归档/删除
- 合规认证：符合《儿童个人信息网络保护规定》
