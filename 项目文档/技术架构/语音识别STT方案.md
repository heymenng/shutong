# 伴读书童AI·语音识别（STT）技术选型方案

> **用途**：让孩子说话，书童能听见  
003e **现状**：只有TTS输出（书童说），没有STT输入（书童听）  
003e **目标**：选择最适合儿童场景的STT方案  
003e **日期**：2026年5月30日  

---

## 一、需求分析

### 1.1 儿童STT的特殊挑战

| 挑战 | 成人STT | 儿童STT |
|------|---------|---------|
| **发音清晰度** | 较高 | 较低（换牙期、语言发育中） |
| **词汇量** | 大 | 小（用词简单、重复） |
| **语速** | 稳定 | 变化大（快/慢/停顿多） |
| **噪音环境** | 可控 | 不可控（玩耍、哭闹、背景音乐） |
| **方言/口音** | 多样 | 家庭环境影响大 |
| **情绪影响** | 较小 | 较大（哭时说话不清） |

### 1.2 书童STT的核心要求

1. **实时性**：延迟 < 500ms（对话流畅）
2. **准确率**：中文识别率 > 90%（儿童语音）
3. **离线能力**：断网也能用（隐私+稳定性）
4. **低资源**：能在机器人端侧运行（CPU/内存有限）
5. **唤醒机制**：支持关键词唤醒（"书童"/"你好书童"）
6. **打断机制**：孩子说"停"就停止当前输出

## 二、方案对比

### 方案1：Whisper（OpenAI）

| 维度 | 详情 |
|------|------|
| **模型** | Whisper Large-v3 / Whisper Medium |
| **准确率** | 中文95%+（成人），儿童约85% |
| **延迟** | 云端1-3秒，本地3-10秒 |
| **离线能力** | ✅ whisper.cpp支持本地 |
| **资源占用** | Medium模型约5GB内存 |
| **儿童适配** | ⚠️ 对儿童语音优化一般 |
| **成本** | 云端$0.006/分钟，本地免费 |

**优点**：
- 开源，生态好
- 多语言支持（中英文混合）
- whisper.cpp可在端侧运行

**缺点**：
- 对儿童语音、方言识别率下降
- 大模型端侧运行较慢

**适用场景**：有GPU/较好CPU的端侧设备

---

### 方案2：FunASR（阿里巴巴）

| 维度 | 详情 |
|------|------|
| **模型** | Paraformer / SenseVoice |
| **准确率** | 中文96%+（成人），儿童约88% |
| **延迟** | 云端200ms，本地500ms-2秒 |
| **离线能力** | ✅ 支持本地部署 |
| **资源占用** | Paraformer-small约500MB |
| **儿童适配** | ✅ 有儿童语音数据集优化 |
| **成本** | 阿里云按量计费，本地免费 |

**优点**：
- 中文优化最好
- 支持实时流式识别
- 有情感识别能力（SenseVoice）
- 模型轻量，适合端侧

**缺点**：
- 英文识别较弱
- 部署文档较复杂

**适用场景**：纯中文场景，端侧资源有限

---

### 方案3：百度/讯飞/腾讯API

| 维度 | 百度 | 讯飞 | 腾讯 |
|------|------|------|------|
| **准确率** | 中文92% | 中文94% | 中文91% |
| **延迟** | 200-500ms | 100-300ms | 200-500ms |
| **离线能力** | ❌ 仅云端 | ❌ 仅云端 | ❌ 仅云端 |
| **儿童适配** | ✅ 有儿童模式 | ✅ 有儿童模式 | ⚠️ 一般 |
| **成本** | ¥0.015/分钟 | ¥0.02/分钟 | ¥0.018/分钟 |

**优点**：
- 即开即用，无需部署
- 有专门的儿童语音识别优化
- 稳定， SLA保障

**缺点**：
- 必须联网
- 有费用
- 数据上传云端，隐私风险

**适用场景**：快速原型、网络稳定环境

---

### 方案4：自研/微调模型

| 维度 | 详情 |
|------|------|
| **基础模型** | Whisper-small / Wav2Vec2 |
| **微调数据** | 儿童语音数据集（需要收集） |
| **准确率** | 理论上可达90%+ |
| **延迟** | 取决于模型大小 |
| **离线能力** | ✅ 完全本地 |
| **资源占用** | small模型约500MB-1GB |

**优点**：
- 完全可控
- 可针对书童场景优化
- 完全离线，隐私最好

**缺点**：
- 需要数据收集和标注
- 需要AI工程师
- 周期长（3-6个月）

**适用场景**：长期产品，有技术团队

## 三、推荐方案

### 阶段性方案

```
第一阶段（Demo/试点）：云端API（讯飞/百度）
  ├── 优点：快速上线，识别率高
  ├── 缺点：依赖网络，有费用
  └── 成本：¥0.015-0.02/分钟

第二阶段（产品化）：FunASR端侧部署
  ├── 优点：中文最好，可离线，轻量
  ├── 缺点：部署复杂，英文弱
  └── 资源：500MB-1GB内存

第三阶段（长期）：自研微调模型
  ├── 优点：完全可控，场景最优
  ├── 缺点：周期长，需要数据
  └── 周期：3-6个月
```

### 最终推荐：FunASR（Paraformer-small）

**理由**：
1. 中文识别率最高（儿童语音优化）
2. 模型轻量（500MB），适合机器人端侧
3. 支持流式识别（实时对话）
4. 完全离线，保护隐私
5. 开源免费，无授权费用
6. 支持情感识别（SenseVoice可辅助判断情绪）

## 四、技术实现

### 4.1 系统架构

```
麦克风输入
  ↓
【VAD（语音活动检测）】
  - 过滤静音/噪音
  - 检测语音起止
  ↓
【唤醒词检测】
  - "书童"/"小书童"/"你好书童"
  - 本地运行，低功耗
  - 检测到唤醒 → 进入对话模式
  ↓
【FunASR识别】
  - 流式识别（边说边转文字）
  - 输出文字送入书童AI
  ↓
【书童AI处理】
  - 生成回复
  - TTS输出
  ↓
【打断检测】
  - 孩子说"停"/"等一下"
  - 立即停止当前输出
```

### 4.2 唤醒词设计

| 唤醒词 | 适用场景 | 优先级 |
|--------|---------|--------|
| "书童" | 日常唤醒 | P0 |
| "小书童" | 亲切称呼 | P1 |
| "你好书童" | 正式唤醒 | P1 |
| "书童在吗" | 确认在线 | P2 |

**唤醒词优化**：
- 支持方言发音（普通话/粤语/四川话）
- 支持远场唤醒（3米内）
- 误唤醒率 < 1次/24小时

### 4.3 打断机制

| 打断词 | 动作 | 说明 |
|--------|------|------|
| "停" | 立即停止TTS输出 | 最高优先级 |
| "等一下" | 暂停，等待继续 | 保留上下文 |
| "我不想听了" | 停止，询问原因 | 情绪识别触发 |
| "换个话题" | 停止，切换话题 | 上下文重置 |

### 4.4 噪音处理

| 噪音类型 | 处理方式 | 技术 |
|---------|---------|------|
| 环境噪音（空调/电视） | 降噪滤波 | RNNoise/WebRTC AEC |
| 多人说话 | 声源分离 | Beamforming |
| 孩子跑动/玩耍 | 运动检测 | 加速度传感器辅助 |
| 背景音乐 | 音乐检测 | 频谱分析 |

## 五、性能指标

| 指标 | 目标值 | 测试方法 |
|------|--------|---------|
| **识别延迟** | < 500ms | 从说完到文字输出 |
| **识别准确率** | > 90% | 100句儿童语音测试 |
| **唤醒成功率** | > 95% | 100次唤醒测试 |
| **误唤醒率** | < 1次/天 | 24小时静默测试 |
| **打断响应** | < 200ms | 说"停"到停止输出 |
| **CPU占用** | < 30% | 机器人端侧运行 |
| **内存占用** | < 1GB | 模型+运行开销 |

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 儿童发音不清 | 高 | 识别率下降 | 收集儿童语音数据微调 |
| 方言识别差 | 中 | 部分地区无法使用 | 支持主流方言模型 |
| 端侧性能不足 | 中 | 延迟高/卡顿 | 降级到小模型，云端backup |
| 隐私合规 | 中 | 法律风险 | 优先端侧方案，不上传原始语音 |
| 网络不稳定 | 高 | 云端方案失效 | 必须有离线fallback |
